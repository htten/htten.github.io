# 2026-03-02 工作日志

## GitHub Pages + Hexo 博客部署

### 问题诊断
- GitHub Pages 显示 "Page not found"
- 原因：Hexo 编译后的静态文件在 `public/` 目录，但被 Jekyll 处理时出错
- `.nojekyll` 文件可禁用 Jekyll

### 解决方案
1. 创建 Hexo 项目初始化
2. 配置 GitHub Actions 自动部署
3. 推送源码到 main 分支，Actions 自动生成静态文件

### 关键配置
- `.github/workflows/deploy.yml` - 自动部署 workflow
- `_config.yml` - Hexo 配置（主题 language: zh-CN）
- `source/_posts/` - 博客文章目录

### GitHub Token
- 使用 Personal Access Token (fine-grained) 推送代码
- Token 权限：contents: write

### 经验教训
- GitHub API 不支持直接修改 Pages 源码路径（只能选 / 或 /docs）
- 需要把静态文件推送到 main 分支根目录
- GitHub Actions 自动部署最省事
